# 完善资产管理页面计划

## 当前状态分析

资产管理页面目前存在以下问题：
1. 角色/场景/道具的显示和删除功能已添加，但缺少历史图和主图选择功能
2. 预览功能较为简单，缺少轮播浏览
3. 资产编辑功能不完善

## 目标

将资产管理页面完善为与分镜绘制页类似的功能，包括：
1. 历史图管理（保存生成的所有图片）
2. 主图选择功能
3. 图片轮播预览
4. 更好的资产编辑界面

## 实施步骤

### 步骤1: 添加历史图状态管理
**文件**: `src/pages/AssetManage.tsx`

需要添加的状态：
- `imageHistory: string[]` - 存储资产的历史图片
- `currentImageIndex: number` - 当前显示的图片索引
- `mainImageIndex: number` - 主图索引

为每个资产类型（角色、场景、道具）分别管理历史图。

### 步骤2: 创建资产预览轮播组件
**文件**: 复用 `src/components/media/ImageCarousel.tsx`

AssetManage 页面将使用与 StoryboardDraw 相同的 ImageCarousel 组件。

### 步骤3: 修改资产数据结构支持历史图
**文件**: `src/pages/AssetManage.tsx`

需要：
1. 加载资产时同时加载历史图（从 metadata 中读取）
2. 保存资产时保存历史图到 metadata
3. 添加历史图管理函数（添加、删除、设为主图）

### 步骤4: 重新设计资产详情界面
**文件**: `src/pages/AssetManage.tsx`

参考 StoryboardDraw 的布局：
```
左侧（45%宽度）：
- 标签页：预览 / 历史图 / 参考图
- 预览区使用 ImageCarousel 组件
- 历史图网格展示

右侧（55%宽度）：
- 资产信息编辑（名称、描述）
- AI生成控制（模型选择、提示词、生成参数）
- 生成按钮
```

### 步骤5: 添加资产AI生成功能
**文件**: `src/pages/AssetManage.tsx`

复制 StoryboardDraw 的生成逻辑：
1. 添加 `handleGenerate` 函数
2. 添加 `handleGenerateComfyUI` 函数
3. 生成完成后自动添加到历史图
4. 支持设置主图

### 步骤6: 更新资产卡片交互
**文件**: `src/pages/AssetManage.tsx`

修改 AssetCard 和 CharacterCard：
1. 点击卡片打开详情编辑界面
2. 显示主图缩略图
3. 显示历史图数量标记

### 步骤7: 持久化历史图数据
**文件**: `src/db/index.ts` 或 `src/pages/AssetManage.tsx`

历史图存储方案：
- 将历史图数组存储在资产的 metadata 字段中
- 格式：`metadata: { imageHistory: string[], mainImageIndex: number }`
